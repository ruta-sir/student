## Project Overview
 
Wayfinding World is a Canvas RPG-based learning platform built for students moving through the CS Pathway. My contributions span three trimesters and two distinct backend stacks.
 
**Trimester 1–2 (Spring/Flask backend work):**  
I built the **Microblog Groups** feature — a full-stack social posting system where users can post as a member of a group rather than as an individual. This included extending the database schema, building Flask API endpoints, integrating with the existing Spring groups API via JWT authentication, and building a frontend post composer with context-aware routing.
 
**Trimester 3 (Wayfinding World capstone):**  
I built the **Skill Passport** — a full-screen passport-themed modal that fetches a student's skill snapshot history from a Python backend, renders their current skill levels (Coding, Collaboration, Problem Solving, Initiative) with progress bars, and displays a chronological history of past snapshots styled as stamped passport entries. The `SkillPassport` is implemented as a standalone class that handles its own fetching, rendering, and DOM cleanup.
 
The broader platform includes Identity Forge (persona selection), a personality quiz, Code Hub (HTML learning), Sprint Success (agile missions), and NPC-guided navigation between zones.

## Personal & Social Relevance
 
### Real-World Problem Addressed
Two problems motivated my work:
 
1. **Group posting without identity loss** — students in a class wanted to post collaboratively under a group name without losing their individual identity. My Microblog Groups feature solved this by adding an `author_type` field that switches post attribution between user and group, while keeping permission checks to prevent unauthorized group posting.
2. **Invisible skill growth** — students complete projects but rarely see their own progression clearly. The Skill Passport converts backend skill snapshot data into a visual, passport-themed record of growth — reinforcing the idea that learning is a journey with real milestones. My mom works on similar data aggregation systems professionally, which motivated me to build this more seriously.
### Ethical Considerations
- **Privacy:** All skill and persona data is gated behind JWT cookie authentication. No student data is readable without a valid session.
- **Security:** Group post endpoints enforce multi-stage permission checking — 401 (unauthenticated), 403 (no group permission), 404 (invalid group) — so no user can post as a group they don't belong to.
- **Equity:** Skill scores use a 1–6 scale with progress bars rather than letter grades, reducing the stigma of low scores and emphasizing growth over rank.
- **Accessibility:** The Skill Passport modal uses high-contrast colors (dark green, gold) and is keyboard-closeable. NPC dialogue in the game world provides text-based guidance requiring no prior game experience.

## Data Structures
 
### 1. Graph (Adjacency List)
**Where used:** Core wayfinding engine — hub zones as nodes, transitions as edges  
**Why:** Adjacency list is memory-efficient for sparse hub graphs where most zones connect to only 2–3 others, not every zone to every other
 
```java
// Replace with your actual code
Map<Integer, List<Edge>> adjacencyList = new HashMap<>();
 
public void addEdge(int from, int to, double weight) {
    adjacencyList.computeIfAbsent(from, k -> new ArrayList<>())
                 .add(new Edge(to, weight));
}
```
**Big-O:** O(1) amortized edge insertion; O(degree) neighbor traversal — efficient for sparse graphs.

### 4. Queue (BFS + Level Injection)
**Where used:** BFS shortest-path traversal; game level injection pattern (sub-levels spliced dynamically)  
**Why:** FIFO ordering guarantees BFS explores nodes level-by-level for shortest path; level injection relies on ordered processing after index-based splice
 
```java
// Replace with your actual code
Queue<Integer> queue = new LinkedList<>();
queue.offer(startId);
while (!queue.isEmpty()) {
    int current = queue.poll();
    for (Edge e : adjacencyList.getOrDefault(current, new ArrayList<>())) {
        if (!visited.contains(e.getTo())) queue.offer(e.getTo());
    }
}
```
 
> **Engineering Note:** Splicing a sub-level at `currentLevelIndex + 1` then incrementing is the correct pattern — it inserts without corrupting existing level order.

## Algorithms
 
### 1. BFS Pathfinding — Hub Navigation
**Purpose:** Find the shortest unweighted path between two hub zones (e.g., Identity Forge → Code Hub)  
**Time Complexity:** O(V + E) — each node and edge visited at most once  
**Space Complexity:** O(V) — queue and visited set each hold at most V entries
 
```java
// Replace with your actual implementation
public List<Location> findPath(int startId, int endId) {
    Queue<Integer> queue = new LinkedList<>();
    Map<Integer, Integer> cameFrom = new HashMap<>();
    Set<Integer> visited = new HashSet<>();
 
    queue.offer(startId);
    visited.add(startId);
 
    while (!queue.isEmpty()) {
        int current = queue.poll();
        if (current == endId) break;
        for (Edge e : adjacencyList.getOrDefault(current, new ArrayList<>())) {
            if (!visited.contains(e.getTo())) {
                visited.add(e.getTo());
                cameFrom.put(e.getTo(), current);
                queue.offer(e.getTo());
            }
        }
    }
    return reconstructPath(cameFrom, startId, endId);
}
```
**Why BFS over Dijkstra's?** Hub zone transitions are unweighted (each step costs 1), so BFS guarantees shortest path with lower constant-factor overhead than Dijkstra's priority queue.

### 4. Hashing — Password Security & Efficient Lookups
**Where used:** BCrypt password hashing in authentication; HashMap internal hashing for O(1) persona and UID lookups  
**Why:** Plaintext password storage means a single DB breach exposes every account. BCrypt salts the hash (preventing rainbow table attacks) and applies a configurable cost factor (slowing brute-force). The same hashing principle underlies HashMap's O(1) lookups.
 

## Object-Oriented Design
 
### Abstraction
`Waypoint` is an abstract class defining the contract every navigable entity must fulfill. Subclasses provide the concrete implementation — the game loop never needs to know whether it's dealing with a `Location` or a `Landmark`.
 
```java
// Replace with your actual code
public abstract class Waypoint {
    protected String name;
    protected double x, y;
 
    public abstract String getType();
    public abstract String getDescription();
}
```
 
In JavaScript, `SkillPassport` encapsulates all fetch/render/cleanup logic behind a clean public interface — callers just instantiate it and call `open()`.
 
---
 
### Encapsulation
Fields are private with validated setters. The Skill Passport's score setter rejects out-of-range values internally — callers can't accidentally corrupt the data model.
 
```java
// Replace with your actual code
public class SkillSnapshot {
    private int userId;
    private double codingAbility;   // 1–6 scale
 
    public double getCodingAbility() { return codingAbility; }
    public void setCodingAbility(double score) {
        if (score < 1 || score > 6)
            throw new IllegalArgumentException("Score must be 1–6");
        this.codingAbility = score;
    }
}
```

### Inheritance
`Landmark` extends `Location`, inheriting position and name while adding a `category` field for named points of interest (NPC positions, teleport gates, zone entrances).
 
```java
// Replace with your actual code
public class Landmark extends Location {
    private String category; // "hub", "gate", "npc", "exit"
 
    @Override
    public String getDescription() {
        return "[" + category.toUpperCase() + "] " + super.getName();
    }
}
```

### Polymorphism
The game loop iterates a `List<Waypoint>` containing mixed `Location` and `Landmark` instances. `getDescription()` dispatches to the correct subclass at runtime without any type-checking.
 
```java

waypoints.add(new Landmark("Identity Forge", 100, 200, "hub"));
waypoints.add(new Location("Hallway Node A", 150, 200));

}
```

### Debugging Process
 
> **Bug (Tri 3):** Skill Passport UI rendered completely blank despite the backend running correctly  
> **How I found it:** Wrapped the `fetch()` in `try/catch`, added `if (!res.ok) throw new Error(res.status)` before parsing JSON, and logged the status — it was returning 404 due to a trailing slash mismatch (`/api/user/skill-snapshot/` vs `/api/user/skill-snapshot`)  
> **Fix:** Corrected the endpoint path and standardized all fetch calls to check `res.ok` before parsing. Silent path mismatches are now impossible to miss.
 
> **Bug (Tri 2):** GitHub commits weren't being attributed to my account  
> **How I found it:** Noticed missing contribution history in GitHub analytics  
> **Fix:** Identified an email configuration mismatch in `git config`, updated it, and verified subsequent commits appeared correctly

## Help System
 
### How to Use Wayfinding World
 
1. **Log in** with your student credentials at the home screen
2. **Enter Identity Forge** — choose your persona archetype from four categories (Student, Social, Achievement, Fantasy). This drives personalization across the platform.
3. **Travel to Wayfinding World** via the teleport gate that unlocks after persona selection
4. **Complete the Personality Quiz** — NPC gatekeepers guide you through each section. Results go to your teacher's grouping dashboard.
5. **Explore Mission Tools** — Code Hub for HTML coding challenges; Sprint Success for agile planning missions. Skills earned are logged to your Skill Passport automatically.
6. **Open your Skill Passport** from the main hub at any time to view your current skill levels (Coding, Collaboration, Problem Solving, Initiative) and your full snapshot history across all projects.

## Retrospective
 
### What Went Well
- The `SkillPassport` standalone class pattern worked cleanly — easy to drop into any game level with no coupling
- Learning both Flask and Spring gave me a real understanding of how the two frameworks compare and how to connect them; my mom works on similar cross-system integrations professionally, which helped me see the real-world relevance

The Skill Passport project specifically pushed me to think about how to convert raw backend data into something visually meaningful — not just that it works, but that students can see themselves in it. Building with both Flask and Spring gave me perspective on REST API design that I wouldn't have gotten from either alone.
